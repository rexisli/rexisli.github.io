📘 算法原理与正确性证明

1. 核心思想：离线分块与状态复用

莫队算法是一种离线区间查询的暴力优化算法。其核心在于：

  • 将所有查询按特定规则排序，使得相邻查询的区间重叠度尽可能高。
  • 通过移动左右端点指针 L 和 R，利用上一个查询的区间状态，以 O(1) 的代价转移到当前查询区间。
  • 本质是根号分治在查询排序上的应用。

2. 排序策略与块大小选取证明

设原数组长度为 n，查询次数为 m。将区间 [1, n] 划分为大小为 B 的若干块。
排序规则为：优先按 l / B 升序，同块内按 r 升降序交替。

指针移动代价分析：

  • 左指针 L：在同块内移动，每次最多移动 B 步，m 次查询总代价 O(m·B)。跨块时最多跳跃 n 步，总跨块代价 O(n)。
  • 右指针 R：在同块内，r 单调变化（奇偶优化使其呈“蛇形”往返），每块最多遍历整个数组 O(n)。共 n/B 块，总代价 O(n²/B)。

总移动次数：T(B) = O(m·B + n + n²/B)。
由均值不等式，当 m·B ≈ n²/B 即 B = n/√m 时取得最小值。若 n ≈ m，则 B = √n，此时 T = O(n√n)。
✅ 结论：块大小取 √n（或原代码的 √m，在 n≈m 时等价）可使总转移次数最优。

3. 奇偶块优化（蛇形排序）原理

传统莫队同块内 r 始终升序。当处理完第 k 块（r 停在 n）切换至第 k+1 块时，r 需从 n 暴力跳回 1，产生 O(n) 的冗余移动。
奇偶优化：奇数块 r 升序，偶数块 r 降序。切换块时，r 的起点与下一块的起点天然靠近，避免了“折返跑”。
效果：常数级优化，实际运行时间通常可缩短 30%~50%，且不改变渐进复杂度。

4. 时间复杂度严格证明：O((n+m)√n)

  • 排序复杂度：O(m log m)，通常远小于转移复杂度。
  • 转移复杂度：如上所述，L 移动 O(m√n + n)，R 移动 O(n√n)。
  • 单次转移代价：add/del 均为 O(1)。
  • 总时间复杂度：O(m log m + n√n)。在算法竞赛中通常简记为 O(n√n)。
  • 空间复杂度：O(n + m) 用于存储原数组、频次数组、查询与答案。

5. add/del 操作的数学等价性

目标函数为 cur = Σ_{v} c[v]²（各数值频次平方和）。
当加入一个值为 x = a[p] 的元素时：

代码实现采用 cur -= c[x]²; c[x]++; cur += c[x]²;，代数展开后与 cur += 2·c[x] + 1 完全等价。该写法不依赖推导，直接基于定义，避免公式记忆错误，且对任何可维护函数 f(c)（如 f(c)=c, f(c)=c³）均通用，只需替换平方即可。

6. 指针移动顺序与边界安全

代码采用 先扩后缩 顺序：

  • 正确性：初始 L=1, R=0 表示空区间。先扩大区间保证 add 操作始终在合法范围内执行，再收缩移除多余元素。若顺序颠倒（先缩后扩），可能导致区间非法（如 L>R）或重复计算。
  • 越界安全：题目保证 1 ≤ l ≤ r ≤ n，且数组 a[] 为 1-indexed，指针移动严格在 [1, n] 内，无需额外边界判断。

✅ 使用提示：

  1. 该模板适用于可离线且区间转移代价为 O(1) 的静态区间查询问题（如区间众数、区间不同数、区间平方和等）。
  2. 若需支持带修改的莫队，需引入第三维时间轴，块大小调整为 n^(2/3)，复杂度升至 O(n^(5/3))。
  3. 实际竞赛中建议将 a[], c[], n, m 移至全局作用域，并确保 ans 与 cur 使用 long long 防溢出。
