📘 算法原理与正确性证明

1. 预处理的作用

  • 统一奇偶性：原字符串中，奇数长度回文中心在字符上，偶数长度回文中心在字符间隙。插入 # 后，所有回文中心都落在字符上，且回文长度恒为奇数。
  • 哨兵防越界：~ 和 $ 是任意两个不相同且不在原串中出现的字符。在 while(s[i-p[i]] == s[i+p[i]]) 扩展时，一旦触达边界必然不相等，自动终止循环，省去 i-p[i]>=0 && i+p[i]<len 的判断开销。

2. 核心递推公式证明：p[i] = min(p[2*C-i], R-i)

设当前遍历到位置 i，且 i < R。令 j = 2*C - i 为 i 关于 C 的对称点。已知以 C 为中心的回文串覆盖区间为 [C-(R-C), R-1]（注意 R 是右边界下一位置）。根据 p[j] 与左边界 C-(R-C) 的关系，分三种情况：

情况 几何关系 结论 证明
① 完全包含 p[j] < R-i p[i] = p[j] j 的回文完全在 C 的回文内部，由回文对称性，i 的回文与 j 完全对称，无需扩展。
② 恰好贴边 p[j] == R-i p[i] ≥ R-i，需继续扩展 j 的回文左端恰好触及 C 回文左边界。对称后 i 的回文右端至少到 R，但 R 以外的字符未知，必须从 R 开始继续 while 扩展。
③ 超出左界 p[j] > R-i p[i] = R-i，无需扩展 j 的回文超出左边界，说明在 R 之外必有字符破坏了对称性（否则 C 的回文能更靠右）。因此 i 最多只能到 R，直接取 R-i。

综上：无论哪种情况，p[i] 的初始安全下界都是 min(p[j], R-i)。算法先赋予该值，再通过 while 处理情况②的后续扩展，保证了正确性且不漏掉任何可能。

3. 时间复杂度证明：O(N)

  • R 是单调不减的。每次 while 循环成功匹配一次，意味着 s[i+p[i]] 与 s[i-p[i]] 相等，此时 i+p[i] 会尝试突破当前的 R。
  • 一旦突破，R 就会被更新为更大的值。由于 R 最大只能达到 len，整个算法过程中 while 循环的成功比较次数最多为 len 次。
  • 初始化赋值、边界判断、更新操作均为 O(1)。
  • 总时间复杂度：O(len) = O(N)，N 为原字符串长度。
  • 空间复杂度：预处理字符串 s 与半径数组 p 均为 O(N)。

4. 索引映射公式推导

预处理后的字符串 s 结构如下（以 input = "abc" 为例）：

  • s 中任意原字符 input[k] 对应的下标为 2k + 2。
  • 设最长回文在 s 中的中心为 ans_index，半径为 p[ans_index]。其左边界（不含哨兵）在 s 中的索引为 L = ans_index - p[ans_index]。
  • 观察上表可知，s 的左边界 L 必为 # 或 ~，其右侧紧邻的原串字符索引恰好为 L / 2。

    例：L=1(#) → 原串起始 1/2=0(a)；L=3(#) → 原串起始 3/2=1(b)

  • 原串回文长度：s 中回文总长为 2*p[ans_index]-1，其中包含 p[ans_index] 个 # 和 p[ans_index]-1 个原字符。故原串长度为 p[ans_index]-1。
